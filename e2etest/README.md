# E2E тесты Kanban

## Запуск

```bash
npm test              # headless, все тесты
npm run test:headed   # с открытым браузером
npm run test:ui       # Playwright UI mode
npm run report        # открыть HTML-отчёт
```

Тесты запускаются против `http://localhost:80` (Docker Compose).

## Структура

```
specs/
  auth.spec.ts        — регистрация, логин, навигация
  project.spec.ts     — создание, обновление, удаление проекта
  board.spec.ts       — просмотр доски, колонки, swimlanes
  task.spec.ts        — CRUD задач, перемещение, архив
  comment.spec.ts     — создание, редактирование, удаление комментариев
  document.spec.ts    — загрузка и удаление документов
  file.spec.ts        — прикрепление файлов к задачам
  search.spec.ts      — поиск задач, фильтры, пагинация
  report.spec.ts      — отчёты CFD и Lead Time
  access.spec.ts      — управление группами доступа
  realtime.spec.ts    — SSE-синхронизация между пользователями (пропущен)
  example.spec.ts     — пример smoke-теста
helpers.ts            — вспомогательные функции (логин, создание сущностей)
fixtures.ts           — расширенный `test` с авто-мониторингом 5xx и console.error
```

## Тестовые сценарии

### Auth (auth.spec.ts)

**Навигация:**
1. Открыть `/login` → кликнуть «Register» → проверить URL `/register` и заголовок «Register»
2. Открыть `/register` → кликнуть «Login» → проверить URL `/login` и заголовок «Login»
3. Открыть `/` → проверить редирект на `/login` и заголовок «Login»

**Регистрация:**
1. Открыть `/register` → заполнить имя, email, пароль → нажать «Register» → проверить URL `/login`

**Логин:**
1. Зарегистрировать пользователя → перейти на `/login` → ввести email/пароль → нажать «Login» → проверить URL `/projects`

### Project (project.spec.ts)

**Создание:**
1. Открыть `/projects` → нажать «New project» → заполнить имя → нажать «Create» → проверить элемент списка и текст названия

**Обновление имени:**
1. Создать проект → перейти в его настройки → изменить имя → нажать «Save» → проверить `input` с новым именем

**Удаление:**
1. Создать проект → открыть настройки → подтвердить диалог удаления → нажать «Delete project» → проверить URL `/projects`

### Board (board.spec.ts)

**Просмотр:**
1. Создать проект и доску через API → открыть `/boards/{id}` → проверить заголовок и название доски

**Кнопка «Add column»:**
1. Создать проект и доску → открыть доску → проверить видимость кнопки «Add column»

**Swimlanes toggle:**
1. Создать проект и доску → открыть доску → кликнуть toggle swimlanes → проверить `aria-pressed="true"` → кликнуть ещё раз → проверить `aria-pressed="false"`

### Task (task.spec.ts)

**Создание через модалку:**
1. Создать проект, доску → открыть доску → нажать «+» в колонке → заполнить название в модалке → нажать «Create» → проверить отображение задачи

**Открытие деталей:**
1. Создать задачу → кликнуть на карточку → проверить URL `/tasks/{id}` и название задачи

**Обновление:**
1. Создать задачу → открыть детали → нажать «Edit» → изменить название → нажать «Save» → проверить новое название

**Перемещение между колонками:**
1. Создать задачу → нажать «Move» → выбрать вторую колонку в меню → проверить отображение задачи

**Архивация:**
1. Создать задачу → нажать «Archive» → проверить, что задача исчезла

**Удаление с подтверждением:**
1. Создать задачу → нажать «Delete» → нажать «Yes» → проверить, что задача исчезла

### Comment (comment.spec.ts)

**Создание:**
1. Создать задачу → открыть `/tasks/{id}` → ввести текст комментария → нажать «Post» → проверить отображение текста

**Редактирование:**
1. Создать комментарий → нажать «Edit» → изменить текст → нажать «Save» → проверить обновлённый текст

**Удаление:**
1. Создать комментарий → нажать «Delete» → проверить, что текст исчез

### Document (document.spec.ts)

**Навигация:**
1. Создать проект → открыть `/projects/{id}/documents` → проверить заголовок «Documents» и кнопку «Upload»

**Открытие модалки:**
1. Перейти на страницу документов → нажать «Upload» → проверить видимость диалога и текста «Upload document»

**Загрузка:**
1. Открыть модалку → заполнить название → выбрать файл → нажать «Upload» в диалоге → проверить таблицу документов

**Пустое состояние:**
1. Перейти на страницу документов → проверить текст «No documents»

**Удаление:**
1. Загрузить документ → подтвердить диалог → нажать кнопку удаления → проверить текст «No documents»

### File (file.spec.ts)

**Секция файлов на задаче:**
1. Создать задачу → открыть `/tasks/{id}` → проверить видимость секции `.files` и заголовка «Files»

**Выбор файла через browse:**
1. Открыть задачу → кликнуть `.files__browse` → проверить, что filechooser открыт

_Тесты загрузки и удаления файлов пропущены — компонент `FileUpload.vue` содержит заглушку «Real upload to MinIO is not yet implemented»._

### Search (search.spec.ts)

**Навигация:**
1. Открыть `/search` → проверить заголовок «Search Tasks» и поле ввода

**Кнопка «Clear filters»:**
1. Открыть `/search` → проверить видимость кнопки «Clear filters»

**Секция фильтров:**
1. Открыть `/search` → проверить видимость секции `.search-page__filters` и заголовка «Filters»

**Пустой результат:**
1. Открыть `/search` → ввести заведомо отсутствующий запрос → проверить текст «No results»

**Разворачивание фильтров:**
1. Открыть `/search` → кликнуть `<summary>` → проверить наличие 7 полей в `.search-page__filter-grid`

**Пагинация:**
1. Открыть `/search` → ввести «a» → если пагинация видна, проверить `.search-page__page-info`

### Report (report.spec.ts)

**Навигация:**
1. Открыть `/reports` → проверить заголовок «Reports», кнопки «CFD Chart» и «Lead Time»

**CFD по умолчанию:**
1. Открыть `/reports` → проверить активный таб CFD

**Фильтры CFD:**
1. Открыть `/reports` → проверить поля Project ID, Board ID, From, To, Interval

**Кнопка Load:**
1. Открыть `/reports` → проверить видимость кнопки «Load»

**Переключение на Lead Time:**
1. Открыть `/reports` → нажать «Lead Time» → проверить активный таб Lead Time

**Фильтры Lead Time:**
1. Открыть `/reports` → переключиться на Lead Time → проверить поля Project ID, From, To

**Изменение дат:**
1. Открыть `/reports` → заполнить два `input[type="date"]`

**Состояние загрузки (пропущен — тест без assert, бэкенд CFD возвращает 500):**
1. Открыть `/reports` → нажать «Load» → подождать 300 мс

### Access (access.spec.ts)

**Навигация:**
1. Открыть `/access` → проверить заголовок «Управление доступом» и кнопку «Создать группу»

**Создание группы:**
1. Открыть `/access` → нажать «Создать группу» → заполнить имя → отправить → проверить `.access-control__group-item`

**Выбор группы:**
1. Создать группу → кликнуть по ней → проверить `.access-control__main`

**Добавление участника (пропущен — 500 на бэкенде):**
1. Создать группу → выбрать её → ввести ID пользователя → нажать кнопку → проверить `.access-control__member-item`

**Вкладка «Разрешения»:**
1. Создать группу → выбрать её → нажать «Разрешения» → проверить текст «Разрешения группы»

**Удаление группы:**
1. Создать группу → нажать «Delete» → подтвердить в диалоге → проверить, что группа исчезла

## Соглашения

- **Теги:** `@smoke @critical` — критические сценарии для быстрой проверки; `@regression` — полная регрессия.
- **Авто-мониторинг:** `fixtures.ts` перехватывает `console.error` и ответы с HTTP 5xx; после каждого теста проверяется их отсутствие.
- **Изоляция:** каждый тест создаёт уникальные данные через `Date.now().toString(36)`.
- **Skipped-тесты:** отключены через `test.skip` или `test.fixme` с комментарием причины.
- **Dialog:** для подтверждения `window.confirm` используется `page.once('dialog', dialog => dialog.accept())`.
