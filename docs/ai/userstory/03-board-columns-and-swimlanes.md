# Пользовательские сценарии: Колонки и свимлайны

<a id="us-03-01"></a>
## US-03-01: Создание колонки
**Роль:** Участник проекта с правом редактирования доски  
**Цель:** Добавить новую колонку статуса  
**Сценарий:**
1. Пользователь нажимает «Добавить колонку».
2. Вводит название колонки.
3. Система создаёт пустую колонку справа от существующих.

**Связанный сценарий использования:** [UC-03-01](../usecase/03-board-columns-and-swimlanes.md#uc-03-01)

<a id="us-03-02"></a>
## US-03-02: Редактирование колонки
**Роль:** Участник проекта с правом редактирования доски  
**Цель:** Переименовать колонку  
**Сценарий:**
1. Пользователь открывает меню колонки.
2. Выбирает «Редактировать».
3. Изменяет название.
4. Сохраняет.

**Связанный сценарий использования:** [UC-03-02](../usecase/03-board-columns-and-swimlanes.md#uc-03-02)

<a id="us-03-03"></a>
## US-03-03: Удаление колонки
**Роль:** Участник проекта с правом редактирования доски  
**Цель:** Удалить колонку  
**Сценарий:**
1. Пользователь открывает меню колонки.
2. Выбирает «Удалить».
3. Система запрашивает подтверждение (с предупреждением о потере задач в колонке).
4. Пользователь подтверждает.
5. Система удаляет колонку; задачи переносятся в бэклог.

**Связанный сценарий использования:** [UC-03-03](../usecase/03-board-columns-and-swimlanes.md#uc-03-03)

<a id="us-03-04"></a>
## US-03-04: Открытие/скрытие бэклога
**Роль:** Участник проекта  
**Цель:** Посмотреть или скрыть бэклог  
**Сценарий:**
1. Пользователь нажимает кнопку бэклога слева.
2. Панель бэклога открывается (или закрывается).

**Связанный сценарий использования:** [UC-03-04](../usecase/03-board-columns-and-swimlanes.md#uc-03-04)

<a id="us-03-05"></a>
## US-03-05: Открытие/скрытие архива
**Роль:** Участник проекта  
**Цель:** Посмотреть или скрыть архив  
**Сценарий:**
1. Пользователь нажимает кнопку архива справа.
2. Панель архива открывается (или закрывается).

**Связанный сценарий использования:** [UC-03-05](../usecase/03-board-columns-and-swimlanes.md#uc-03-05)

<a id="us-03-06"></a>
## US-03-06: Автоматическое появление свимлайна (frontend-only)
**Роль:** Участник проекта  
**Цель:** Увидеть новый свимлайн после перемещения задачи с новым Use Case на доску  
**Сценарий:**
1. Пользователь перетаскивает задачу с Use Case «Регистрация» из бэклога в колонку.
2. Фронт группирует задачи в колонке по полю Use Case и обнаруживает, что Use Case «Регистрация» отсутствует среди текущих свимлайнов.
3. Фронт создаёт новый свимлайн «Регистрация» и отображает задачу в нём.

**Связанный сценарий использования:** [UC-03-06](../usecase/03-board-columns-and-swimlanes.md#uc-03-06)

<a id="us-03-07"></a>
## US-03-07: Автоматическое удаление свимлайна (frontend-only)
**Роль:** Участник проекта  
**Цель:** Увидеть, что свимлайн исчез после перемещения последней задачи  
**Сценарий:**
1. Пользователь перетаскивает последнюю задачу из колонки в архив.
2. Фронт перегруппировывает задачи в колонках по полю Use Case и обнаруживает, что задач с данным Use Case не осталось.
3. Фронт убирает пустой свимлайн с доски.

**Связанный сценарий использования:** [UC-03-07](../usecase/03-board-columns-and-swimlanes.md#uc-03-07)
