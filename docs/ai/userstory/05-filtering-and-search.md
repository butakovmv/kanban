# Пользовательские сценарии: Фильтрация и поиск

<a id="us-05-01"></a>
## US-05-01: Фильтрация задач по признаку
**Роль:** Участник проекта  
**Цель:** Отфильтровать задачи на доске  
**Сценарий:**
1. Пользователь открывает панель фильтров.
2. Выбирает значение одного или нескольких признаков (приоритет, исполнитель, метка, Use Case).
3. Система отображает только задачи, соответствующие фильтру, во всех видимых колонках.
4. Если бэклог или архив открыты, фильтр применяется и к ним.

**Связанный сценарий использования:** [UC-05-01](../usecase/05-filtering-and-search.md#uc-05-01)

<a id="us-05-02"></a>
## US-05-02: Фильтр по исполнителю через панель исполнителей
**Роль:** Участник проекта  
**Цель:** Увидеть задачи конкретного исполнителя  
**Сценарий:**
1. Пользователь нажимает на исполнителя в нижней панели.
2. Система применяет фильтр по выбранному исполнителю.
3. Система отображает только задачи этого исполнителя.

**Связанный сценарий использования:** [UC-05-02](../usecase/05-filtering-and-search.md#uc-05-02)

<a id="us-05-03"></a>
## US-05-03: Поиск по задачам
**Роль:** Участник проекта  
**Цель:** Найти задачу по тексту  
**Сценарий:**
1. Пользователь вводит поисковый запрос в поле поиска на доске.
2. Нажимает Enter или кнопку поиска.
3. Открывается новая вкладка с результатами поиска.
4. Система показывает задачи, в названии или описании которых встречается запрос.

**Связанный сценарий использования:** [UC-05-03](../usecase/05-filtering-and-search.md#uc-05-03)

<a id="us-05-04"></a>
## US-05-04: Расширенный поиск с настройками
**Роль:** Участник проекта  
**Цель:** Настроить область поиска  
**Сценарий:**
1. Пользователь вводит поисковый запрос на доске.
2. Перед отправкой настраивает параметры: включить/исключить бэклог, архив, комментарии.
3. Система выполняет поиск с учётом настроек и выводит результаты.

**Связанный сценарий использования:** [UC-05-04](../usecase/05-filtering-and-search.md#uc-05-04)
