# Пользовательские сценарии: Операции с задачами

<a id="us-04-01"></a>
## US-04-01: Создание задачи
**Роль:** Участник проекта  
**Цель:** Добавить новую задачу  
**Сценарий:**
1. Пользователь нажимает кнопку «Новая задача».
2. Заполняет обязательные поля (название) и опциональные (Use Case, приоритет, метки, исполнитель).
3. Система создаёт задачу и помещает её в бэклог.

**Связанный сценарий использования:** [UC-04-01](../usecase/04-task-operations.md#uc-04-01)

<a id="us-04-02"></a>
## US-04-02: Просмотр полной информации о задаче
**Роль:** Участник проекта  
**Цель:** Просмотреть все детали задачи  
**Сценарий:**
1. Пользователь нажимает на карточку задачи.
2. Открывается новая вкладка со страницей задачи.
3. Система отображает полную информацию: название, описание, все даты, исполнителя, приоритет, метки, Use Case, прикреплённые файлы, комментарии, историю изменений.

**Связанный сценарий использования:** [UC-04-02](../usecase/04-task-operations.md#uc-04-02)

<a id="us-04-03"></a>
## US-04-03: Перемещение задачи между колонками через drag-n-drop
**Роль:** Участник проекта  
**Цель:** Изменить статус задачи  
**Сценарий:**
1. Пользователь захватывает карточку задачи и перетаскивает её из колонки «To Do» в колонку «In Progress».
2. Система обновляет статус задачи.
3. Изменение синхронизируется с другими участниками проекта.

**Связанный сценарий использования:** [UC-04-03](../usecase/04-task-operations.md#uc-04-03)

<a id="us-04-04"></a>
## US-04-04: Перемещение задачи через изменение статуса
**Роль:** Участник проекта  
**Цель:** Изменить статус задачи без drag-n-drop  
**Сценарий:**
1. Пользователь открывает полную информацию о задаче.
2. В поле «Статус» выбирает новое значение из выпадающего списка.
3. Система обновляет статус и перемещает задачу в соответствующую колонку.

**Связанный сценарий использования:** [UC-04-04](../usecase/04-task-operations.md#uc-04-04)

<a id="us-04-05"></a>
## US-04-05: Сортировка задач в колонке через drag-n-drop
**Роль:** Участник проекта  
**Цель:** Задать произвольный порядок задач в колонке  
**Сценарий:**
1. Пользователь перетаскивает карточку задачи выше или ниже внутри одной колонки.
2. Система сохраняет новый порядок на сервере.
3. Другие пользователи видят изменённый порядок.

**Связанный сценарий использования:** [UC-04-05](../usecase/04-task-operations.md#uc-04-05)

<a id="us-04-06"></a>
## US-04-06: Редактирование задачи
**Роль:** Участник проекта  
**Цель:** Изменить поля задачи  
**Сценарий:**
1. Пользователь открывает страницу полной информации задачи.
2. Редактирует одно или несколько полей.
3. Сохраняет изменения.
4. Система логирует изменения.

**Связанный сценарий использования:** [UC-04-06](../usecase/04-task-operations.md#uc-04-06)

<a id="us-04-07"></a>
## US-04-07: Добавление комментария к задаче
**Роль:** Участник проекта  
**Цель:** Оставить комментарий к задаче  
**Сценарий:**
1. Пользователь открывает страницу задачи.
2. Вводит текст в поле комментария.
3. Нажимает «Отправить».
4. Комментарий отображается под задачей.

**Связанный сценарий использования:** [UC-04-07](../usecase/04-task-operations.md#uc-04-07)

<a id="us-04-08"></a>
## US-04-08: Прикрепление файла к задаче
**Роль:** Участник проекта  
**Цель:** Прикрепить файл к задаче  
**Сценарий:**
1. Пользователь открывает страницу задачи.
2. Нажимает «Прикрепить файл».
3. Выбирает файл на устройстве.
4. Файл загружается и отображается в списке прикреплённых файлов задачи.

**Связанный сценарий использования:** [UC-04-08](../usecase/04-task-operations.md#uc-04-08)

<a id="us-04-09"></a>
## US-04-09: Архивирование задачи
**Роль:** Участник проекта  
**Цель:** Переместить задачу в архив  
**Сценарий:**
1. Пользователь перетаскивает задачу из колонки в панель архива.
2. Система фиксирует дату архивирования.
3. Задача исчезает из колонки и появляется в архиве.

**Связанный сценарий использования:** [UC-04-09](../usecase/04-task-operations.md#uc-04-09)

<a id="us-04-10"></a>
## US-04-10: Назначение исполнителя
**Роль:** Участник проекта  
**Цель:** Назначить ответственного за задачу  
**Сценарий:**
1. Пользователь открывает карточку задачи или страницу полной информации.
2. В поле «Исполнитель» выбирает пользователя или бота.
3. Изменение сохраняется и отображается на доске.

**Связанный сценарий использования:** [UC-04-10](../usecase/04-task-operations.md#uc-04-10)
