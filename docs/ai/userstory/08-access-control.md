# Пользовательские сценарии: Управление доступом

<a id="us-08-01"></a>
## US-08-01: Создание группы пользователей
**Роль:** Владелец проекта  
**Цель:** Создать новую группу для управления доступом  
**Сценарий:**
1. Владелец открывает раздел «Управление доступом» в проекте.
2. Нажимает «Создать группу».
3. Вводит название группы.
4. Группа создана.

**Связанный сценарий использования:** [UC-08-01](../usecase/08-access-control.md#uc-08-01)

<a id="us-08-02"></a>
## US-08-02: Добавление пользователя в группу
**Роль:** Владелец проекта  
**Цель:** Дать пользователю доступ к проекту  
**Сценарий:**
1. Владелец открывает раздел «Управление доступом».
2. Выбирает группу.
3. Нажимает «Добавить пользователя».
4. Вводит логин или email пользователя.
5. Пользователь добавляется в группу.

**Связанный сценарий использования:** [UC-08-02](../usecase/08-access-control.md#uc-08-02)

<a id="us-08-03"></a>
## US-08-03: Удаление пользователя из группы
**Роль:** Владелец проекта  
**Цель:** Отозвать доступ пользователя к проекту  
**Сценарий:**
1. Владелец открывает раздел «Управление доступом».
2. Выбирает группу.
3. Находит пользователя и нажимает «Удалить».
4. Пользователь теряет доступ к проекту.

**Связанный сценарий использования:** [UC-08-03](../usecase/08-access-control.md#uc-08-03)

<a id="us-08-04"></a>
## US-08-04: Настройка прав доступа группы
**Роль:** Владелец проекта  
**Цель:** Определить, что участники группы могут делать в проекте  
**Сценарий:**
1. Владелец открывает раздел «Управление доступом».
2. Выбирает группу.
3. Настраивает разрешения: просмотр, создание, редактирование, удаление для каждого типа объектов (задачи, документы, колонки).
4. Изменения вступают в силу немедленно.

**Связанный сценарий использования:** [UC-08-04](../usecase/08-access-control.md#uc-08-04)
