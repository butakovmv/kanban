# Пользовательские сценарии: Администрирование

<a id="us-10-01"></a>
## US-10-01: Изменение тарифного плана пользователя
**Роль:** Системный пользователь (администратор)  
**Цель:** Изменить тарифный план пользователя  
**Сценарий:**
1. Администратор находит пользователя в панели управления.
2. Выбирает новый тарифный план.
3. Подтверждает изменение.
4. Новые лимиты применяются к учётной записи пользователя.

**Связанный сценарий использования:** [UC-10-01](../usecase/10-admin.md#uc-10-01)

<a id="us-10-02"></a>
## US-10-02: Просмотр лимитов тарифа
**Роль:** Аутентифицированный пользователь  
**Цель:** Понять, почему нельзя создать ещё один проект или колонку  
**Сценарий:**
1. Пользователь нажимает «Создать проект», но кнопка неактивна или появляется сообщение об ограничении.
2. Система отображает сообщение: «Достигнут лимит по количеству проектов на вашем тарифе».
3. Пользователь переходит в профиль для просмотра тарифного плана.

**Связанный сценарий использования:** [UC-10-02](../usecase/10-admin.md#uc-10-02)
