# Пользовательские сценарии: Совместная работа в реальном времени

<a id="us-09-01"></a>
## US-09-01: Синхронизация изменений между участниками
**Роль:** Два участника проекта, работающих одновременно  
**Цель:** Видеть изменения друг друга без перезагрузки страницы  
**Сценарий:**
1. Пользователь A перемещает задачу из «To Do» в «In Progress».
2. Пользователь B видит это изменение на своей доске с минимальной задержкой.
3. Пользователь B не выполнял никаких действий для обновления.

**Связанный сценарий использования:** [UC-09-01](../usecase/09-realtime.md#uc-09-01)

<a id="us-09-02"></a>
## US-09-02: Конкурирующие изменения
**Роль:** Два участника одновременно редактируют одну задачу  
**Цель:** Не потерять данные при одновременном редактировании  
**Сценарий:**
1. Пользователь A меняет исполнителя задачи.
2. Пользователь B меняет приоритет той же задачи.
3. Оба изменения применяются; система корректно разрешает конфликт.

**Связанный сценарий использования:** [UC-09-02](../usecase/09-realtime.md#uc-09-02)
