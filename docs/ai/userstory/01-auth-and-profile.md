# Пользовательские сценарии: Аутентификация и профиль

<a id="us-01-01"></a>
## US-01-01: Регистрация нового пользователя
**Роль:** Гость  
**Цель:** Создать учётную запись  
**Сценарий:**
1. Гость открывает сайт.
2. Нажимает «Зарегистрироваться».
3. Вводит логин и пароль.
4. Система создаёт учётную запись и присваивает тарифный план по умолчанию.
5. Система перенаправляет пользователя на страницу входа.

**Связанный сценарий использования:** [UC-01-01](../usecase/01-auth-and-profile.md#uc-01-01)

<a id="us-01-02"></a>
## US-01-02: Вход по логину и паролю
**Роль:** Зарегистрированный пользователь  
**Цель:** Войти в систему  
**Сценарий:**
1. Пользователь открывает страницу входа.
2. Вводит логин и пароль.
3. Система аутентифицирует пользователя и перенаправляет на главную страницу.

**Связанный сценарий использования:** [UC-01-02](../usecase/01-auth-and-profile.md#uc-01-02)

<a id="us-01-03"></a>
## US-01-03: Вход с TOTP вместо пароля
**Роль:** Зарегистрированный пользователь с привязанным TOTP  
**Цель:** Войти в систему, используя TOTP вместо пароля  
**Сценарий:**
1. Пользователь открывает страницу входа.
2. Вводит логин.
3. Выбирает способ входа «TOTP» (вместо ввода пароля).
4. Система запрашивает одноразовый код TOTP.
5. Пользователь вводит код из аутентификатора.
6. Система проверяет код и аутентифицирует пользователя.

**Связанный сценарий использования:** [UC-01-03](../usecase/01-auth-and-profile.md#uc-01-03)

<a id="us-01-04"></a>
## US-01-04: Привязка TOTP-аутентификатора
**Роль:** Зарегистрированный пользователь  
**Цель:** Настроить вход по TOTP без пароля  
**Сценарий:**
1. Пользователь переходит в настройки безопасности.
2. Выбирает «Привязать TOTP».
3. Система запрашивает текущий пароль для подтверждения.
4. Пользователь вводит пароль.
5. Система показывает QR-код и секретный ключ.
6. Пользователь сканирует QR-код своим аутентификатором.
7. Пользователь вводит первый код для подтверждения.
8. Система активирует TOTP для учётной записи. Теперь пользователь может входить по логину + TOTP, минуя ввод пароля.

**Связанный сценарий использования:** [UC-01-04](../usecase/01-auth-and-profile.md#uc-01-04)

<a id="us-01-05"></a>
## US-01-05: Восстановление доступа по почте
**Роль:** Гость (забывший пароль)  
**Цель:** Восстановить доступ к учётной записи  
**Сценарий:**
1. Гость нажимает «Забыли пароль?».
2. Вводит адрес электронной почты.
3. Система отправляет ссылку для сброса пароля.
4. Гость переходит по ссылке и задаёт новый пароль.

**Связанный сценарий использования:** [UC-01-05](../usecase/01-auth-and-profile.md#uc-01-05)

<a id="us-01-06"></a>
## US-01-06: Выход из системы
**Роль:** Аутентифицированный пользователь  
**Цель:** Завершить сессию  
**Сценарий:**
1. Пользователь нажимает «Выйти».
2. Система завершает сессию.
3. Пользователь перенаправляется на страницу входа.

**Связанный сценарий использования:** [UC-01-06](../usecase/01-auth-and-profile.md#uc-01-06)

<a id="us-01-07"></a>
## US-01-07: Просмотр тарифного плана
**Роль:** Аутентифицированный пользователь  
**Цель:** Узнать ограничения своего тарифа  
**Сценарий:**
1. Пользователь переходит в настройки / профиль.
2. Система отображает текущий тарифный план и его лимиты.

**Связанный сценарий использования:** [UC-01-07](../usecase/01-auth-and-profile.md#uc-01-07)
