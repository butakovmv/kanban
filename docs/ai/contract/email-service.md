# Контракт: Email Service

## Обзор
Внешний SMTP-сервис для отправки транзакционных писем. Единственный сценарий использования — отправка письма с токеном для восстановления доступа к учётной записи.

## Протокол
- **Протокол:** SMTP over TLS (SMTP+STARTTLS)
- **Порт:** 587 (стандартный submission port)
- **Транспорт:** TCP
- **Аутентификация:** LOGIN или PLAIN (SMTP AUTH)

## Поток вызова

```
┌─────────────┐     SMTP TLS :587      ┌──────────────┐
│ API Container│ ───────────────────→  │ Email Service│
│  (Kotlin)   │ ←───────────────────  │   (внешний)  │
│             │  250 OK / error        │              │
└─────────────┘                        └──────────────┘
```

## Формат письма

### Конверт (SMTP envelope)

| Поле | Значение |
|---|---|
| **MAIL FROM** | `noreply@<domain>` (настраивается, см. конфигурацию) |
| **RCPT TO** | Email пользователя (из запроса `POST /api/v1/auth/recover`) |
| **DATA** | MIME-сообщение (см. ниже) |

### Заголовки (MIME)

| Заголовок | Значение |
|---|---|
| `From` | `Kanban Board <noreply@<domain>>` |
| `To` | `<user email>` |
| `Subject` | `Восстановление доступа к Kanban Board` |
| `MIME-Version` | `1.0` |
| `Content-Type` | `text/html; charset=UTF-8` |
| `X-Mailer` | `Kanban Board <version>` |

### Тело письма (HTML)

```html
<!DOCTYPE html>
<html>
<head><meta charset="UTF-8"></head>
<body>
  <p>Здравствуйте!</p>
  <p>Вы запросили восстановление доступа к учётной записи.</p>
  <p>Для сброса пароля перейдите по ссылке:</p>
  <p><a href="{{recoveryUrl}}">{{recoveryUrl}}</a></p>
  <p>Ссылка действительна в течение 1 часа.</p>
  <p>Если вы не запрашивали восстановление, проигнорируйте это письмо.</p>
</body>
</html>
```

**Переменные шаблона:**
- `{{recoveryUrl}}` — URL вида `https://<frontend-domain>/recover?token=<uuid>` где `<uuid>` — 32-128-символьный токен сброса

**Альтернативная text/plain-часть** (multipart/alternative):

```
Здравствуйте!

Вы запросили восстановление доступа к учётной записи.
Для сброса пароля перейдите по ссылке:
{{recoveryUrl}}

Ссылка действительна в течение 1 часа.
Если вы не запрашивали восстановление, проигнорируйте это письмо.
```

## Параметры отправки

| Параметр | Значение | Примечание |
|---|---|---|
| Таймаут соединения | 10 секунд | `connectionTimeout` |
| Таймаут отправки | 15 секунд | `writeTimeout` |
| Режим аутентификации | STARTTLS | Явное шифрование |
| Проверка сертификата | Да | Если `mail.smtp.ssl.trust` не настроен на `*` |
| Количество попыток | 3 (первая + 2 ретрая) | Exponential backoff: 1с, 3с |

## Обработка ошибок

| Ситуация | Действие сервера | Влияние на API |
|---|---|---|
| SMTP-сервер недоступен | 3 ретрая с backoff, затем лог ERROR | `POST /auth/recover` всё равно возвращает `202 Accepted` |
| Неверные credentials (550 auth fail) | Лог ERROR | `202 Accepted` (нет утечки информации об email) |
| RCPT TO rejected (550) | Лог WARN | `202 Accepted` |
| Таймаут (соединение/отправка) | Ретрий, затем лог WARN | `202 Accepted` |
| Невалидный email (синтаксис) | Отклоняется валидацией до вызова | `400 Bad Request` |

**Важно:** API `POST /api/v1/auth/recover` всегда возвращает `202 Accepted` независимо от результата отправки, чтобы не раскрывать информацию о существовании/несуществовании email в системе.

## Требования к SMTP-серверу

| Требование | Описание |
|---|---|
| STARTTLS | Обязательно (на порту 587) |
| SMTP AUTH | LOGIN или PLAIN |
| Размер сообщения | ≤ 50 KB (письмо ~5-10 KB) |
| Rate limit | ≥ 10 писем/мин (пиковая нагрузка) |
| Доступность | 99.9% (влияет только на восстановление доступа) |

## Конфигурация (application.yml)

```yaml
kanban:
  mail:
    host: "smtp.example.com"
    port: 587
    username: "noreply@example.com"
    password: "${MAIL_PASSWORD}"   # из переменной окружения
    from: "noreply@example.com"
    frontend-url: "https://kanban.example.com"
```

## Безопасность

- Пароль SMTP передаётся через переменную окружения (`MAIL_PASSWORD`), не хранится в репозитории
- TLS-соединение проверяет сертификат (кроме dev-окружения, где `mail.smtp.ssl.trust=*`)
- Токен восстановления — случайный UUID, живёт 1 час, одноразовый
- Токен не передаётся в URL-параметрах письма в открытом виде (закодирован в URL)
