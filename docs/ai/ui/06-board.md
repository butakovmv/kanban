## Функционал

### Шапка доски
- Название проекта
- Поле поиска → открывает `/projects/:id/search` в новой вкладке
- Кнопки перехода к документам, отчётам, управлению доступом

### Панель бэклога (слева, открывается/скрывается)
- Список задач без статуса
- Сортировка по приоритету (более приоритетные выше)
- Drag-n-drop из бэклога в колонки (создаёт задачу на доске)
- Счётчик задач в бэклоге
- Поддержка фильтра (если фильтр активен)

### Колонки (центр)
- Колонки отображаются горизонтально с заголовками
- Drag-n-drop задач между колонками (изменение статуса)
- Drag-n-drop внутри колонки (изменение порядка)
- Индикация количества задач в колонке
- Меню колонки (редактировать, удалить)
- Кнопка «Добавить колонку» справа от всех колонок
- Свимлайны внутри колонок (группировка по Use Case)
  - Заголовок свимлайна (название Use Case)
  - Дефолтный свимлайн «Bug Fix» для задач без Use Case
  - Автоматическое появление/исчезновение свимлайнов

### Карточка задачи (минималистичная)
- Название
- Use Case
- Статус (название колонки)
- Метки (цветные чипсы)
- Исполнитель (аватар)
- Приоритет (цветной индикатор: low/medium/high/critical)
- Drag-n-drop (захват за карточку)
- Клик по карточке → открывает `/projects/:id/tasks/:id` в новой вкладке

### Панель архива (справа, открывается/скрывается)
- Список заархивированных задач
- Сортировка по дате архивирования (более поздние выше)
- Принимает задачи через drag-n-drop из колонок

### Панель исполнителей (снизу, открывается/скрывается)
- Список исполнителей (пользователи + боты)
- Счётчик задач на исполнителя
- Клик по исполнителю → фильтр задач по нему
- Клик по задаче под исполнителем → `/projects/:id/tasks/:id`

### Панель фильтров
- Выбор по: приоритет, исполнитель, метка, Use Case
- Сброс фильтра
- Фильтр применяется к колонкам; к бэклогу и архиву — только если они открыты

### Real-time синхронизация
- WebSocket/SSE-соединение для получения обновлений от других участников
- UI обновляется без перезагрузки страницы
