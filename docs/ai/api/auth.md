# API аутентификации и профиля

Базовый путь: `/api/v1/auth`

## Эндпоинты

### `POST /api/v1/auth/register`
Регистрация нового пользователя.

- **Доступ:** без аутентификации
- **Тело:** `{ "login": "string (3-50, латиница/цифры/_)", "password": "string (8-128)", "email": "email" }`
- **Ответ:** `201 Created` — `{ "userId": "uuid" }`
- **Ошибки:** `409 Conflict` — логин уже занят

### `POST /api/v1/auth/login`
Вход по логину и паролю.

- **Доступ:** без аутентификации
- **Тело:** `{ "login": "string", "password": "string (8-128)" }`
- **Ответ:** `200 OK` — `{ "token": "jwt-string", "user": { "id", "login", "email", "tariff", "hasTOTP": bool } }`
- **Ошибки:** `401 Unauthorized` — неверный логин или пароль

### `POST /api/v1/auth/login/totp`
Вход с TOTP вместо пароля.

- **Доступ:** без аутентификации
- **Тело:** `{ "login": "string", "totpCode": "string (6 цифр)" }`
- **Ответ:** `200 OK` — `{ "token": "jwt-string", "user": { ... } }`
- **Ошибки:** `401 Unauthorized` — неверный код TOTP

### `POST /api/v1/auth/totp/bind`
Привязка TOTP-аутентификатора.

- **Доступ:** аутентифицированный пользователь
- **Тело:** `{ "password": "string (8-128)" }` — подтверждение текущим паролем
- **Ответ:** `200 OK` — `{ "qrCodeUri": "otpauth://...", "secret": "base32..." }`
- **Ошибки:** `401` — не аутентифицирован, `403` — неверный пароль

Затем клиент отправляет первый код для подтверждения:

### `POST /api/v1/auth/totp/verify`
Подтверждение привязки TOTP.

- **Доступ:** аутентифицированный пользователь (после `/totp/bind`)
- **Тело:** `{ "totpCode": "string (6 цифр)" }`
- **Ответ:** `200 OK` — `{ "status": "bound" }`
- **Ошибки:** `401`, `400` — неверный код

### `DELETE /api/v1/auth/totp`
Отвязка TOTP-аутентификатора.

- **Доступ:** аутентифицированный пользователь
- **Тело:** `{ "password": "string" }`
- **Ответ:** `204 No Content`
- **Ошибки:** `401`, `403` — неверный пароль

### `POST /api/v1/auth/recover`
Запрос восстановления доступа.

- **Доступ:** без аутентификации
- **Тело:** `{ "email": "email (макс. 254)" }`
- **Ответ:** `202 Accepted` — всегда одинаковый (даже если email не найден, для защиты от перебора)
- **Side-effect:** сервер отправляет письмо с токеном сброса на указанный email

### `POST /api/v1/auth/recover/reset`
Сброс пароля по токену из письма.

- **Доступ:** без аутентификации (по токену)
- **Тело:** `{ "token": "string (32-128)", "newPassword": "string (8-128)" }`
- **Ответ:** `200 OK`
- **Ошибки:** `400` — неверный или истёкший токен

### `POST /api/v1/auth/logout`
Завершение сессии.

- **Доступ:** аутентифицированный пользователь
- **Заголовки:** `Authorization: Bearer <token>`
- **Ответ:** `204 No Content`

### `GET /api/v1/auth/session`
Информация о текущей сессии.

- **Доступ:** аутентифицированный пользователь
- **Заголовки:** `Authorization: Bearer <token>`
- **Ответ:** `200 OK` — `{ "userId", "login", "email", "tariff": { "name", "limits" } }`

### `GET /api/v1/auth/tariff`
Информация о тарифном плане.

- **Доступ:** аутентифицированный пользователь
- **Ответ:** `200 OK` — `{ "name": "string", "limits": { "maxProjects": 2, "maxColumns": 10, "maxAssignees": 5 } }`
