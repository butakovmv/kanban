Vue 3 SPA
===

**Стек:** Vue 3 (Composition API, `<script setup lang="ts">`), SCSS, TypeScript, Vite 6, Vue Router 4, eslint eslint-plugin-vue, Prettier 

Структура
---
- `index.html` - точка входа в приложение для браузера
- `nginx.conf` - точка обработки статики и вызовов апи для хостинга.
- `src/App.vue` - верстка верхнеуровневого контейнера приложения.
- `src/main.ts` - скрипт инициализации приложения.
- `src/fetch.ts` - конфигурация api-клиента и общая обработка ошибок (нативный `fetch`, обёрнутый в `request<T>(url,options)`, обертки get/post/put/delete).
- `src/pinia.ts` - конфигурация хранилища (pinia).
- `src/router.ts` - конфигурация router.
- `src/style.scss` - общие стили приложения, настройка переменных (цветов) для темной и светлой темы, настройка переменных (размеров, отступов, шрифтов) для разных размеров экрана и ориентаций.
- `src/layout` - глобально переиспользуемые сетки разметки страниц.
- `src/component` - глобально используемые компоненты.
- `src/module/<модуль>/` - модули бизнес логики.

Модули бизнес логики могут содержать специфичные для модуля:
- `*Page.vue` с версткой страницы,
- `*.vue` с версткой компонентов,
- `api.ts` с описанием апи,
- `store.ts` с описанием структур и действий хранилища,
- `*.ts` с выделенной бизнес-логикой операций, изолированной от верстки, апи и хранилищ

Пути и страницы
---
Файлы `docs/ai/ui/##-*.md` описывают страницы, их пути и назначение

Компоненты
---
Файлы `docs/ai/ui/component-*.md` описывают крупные компоненты со сложным поведением, которые могут быть использованы на страницах

**Устройство:** Лэйаут и навигация в `App.vue` (`<router-view />` + `<router-link>`). Маршруты — в `src/router/index.ts`, `createWebHistory()` — nginx уже настроен.

**Тесты:** Vitest + jsdom + Vue Test Utils; `App.spec.ts` (4 теста).

**Сборка:** Vite → `dist/` → `front-dist.tar` → `nginx:alpine` контейнер.

**Линтер:** ESLint v10 flat config + Prettier (без `;`, `singleQuote`).
