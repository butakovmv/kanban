# Модуль usecase
## Назначение
Отвечает за бизнес-логику
Не зависит от других модулей

## Структура
В корневом пакете модуля находятся папки, соотвествующие доменным областям приложения, в этих папках расположены соответствующие доменам файлы.

Содержит публичные интерфейсы операций бизнес-логики, которые может делать приложение. Интерфейсы находятся каждый в отдельном файле, в интерфейсе толко один метод и вложенные интерфейсы входящих аргументов и исходящих результатов операции (при их наличии и не вырожденности). Вместо исключений для сигнализации об альтернативных результатах используются sealed-классы результата, описывающие суть этого результата.

Содержит видимые только внутри модуля реализации этих интерфейсов.

Содержит публичные интерфейсы поставщиков и потребителей данных, которые требуются для этих реализаций.

Все файлы расположены в доменных пакетах 
Принцип именования: интерфейс SomeOperation; метод в нем - простой глагол соответствующий краткой сути операции get/list/send/create/update/remove/notify/save и т.д.; аргумент SomeOperation.Arg и результат SomeOperation.Result с двумя результатами SomeOperation.Result.Success и SomeOperation.Result.Failure

## Проверки
Содерит юнит-тесты реализаций. Интерфейсы поставщиков и потребителей данных мокаются.

Собирается покрытие с помощью jacoco

Статический анализ кода с помощью KtLint и Detekt
