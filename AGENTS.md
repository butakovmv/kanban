# AGENTS.md

## Ограничения и требования от людей.
- Папка `docs/human` содержит вводные от людей. Агенту запрещено редактировать файлы в этой папке.
- `docs/human/SOURCE_CUSTOMER_REQUIRMENT.md` содержит исходные требования человека-заказчика. Агенту запрещено редактировать этот файл.
- `docs/human/SOURCE_TECHNICAL_REQUIRMENT.md` содержит исходные требования человека-архитектора. Агенту запрещено редактировать этот файл.
- `docs/human/SOURCE_UI_UX_REQUIRMENT.md` содержит исходные требования человека-дизайнера. Агенту запрещено редактировать этот файл.

## ADR, история принятых решений
- Папка `docs/adr` содержит описание выработанных решений. Каждый файл должен иметь имя по маске "yyMMdd-hhmm.md" и начинаться со текста "Статус: Разработка" или "Статус: Принято". Агенту запрещено править файлы со статусом "Принято".

## Документация от агента и для агента
- Папка `docs/ai` содержит инструкции для LLM-агентов.
- `docs/ai/FUNCTIONAL_REQUIRMENTS.md` содержит сформированные агентом функциональные требования.
- `docs/ai/NON_FUNCTIONAL_REQUIRMENTS.md` содержит сформированные агентом нефункциональные требования.
- `docs/ai/STRUCTURE.md` содержит описание структуры проекта (от уровня модулей до уровня доменных пакетов для бэкенда или папок для фронтенда).
- `docs/ai/api/<назначение>.md` содержит описание контрактов, - `docs/ai/userstory/*.md` содержит описание пользовательских сценариев с именованием вида US-01-01.
- `docs/ai/contract/<название сторонней системы>.md` содержит описание контрактов взаимодействия приложения со сторонними системами.
- `docs/ai/domain/*.md` содержит описание доменных областей приложения.
предоставляемых приложением пользователю и пользовательскому интерфейсу.
- `docs/ai/dto/<название инструмента>.md` содержит описание контрактов внутренних инструментов (БД, кеши, очереди), с которыми взаимодействует приложение.
- `docs/ai/ui/*.md` содержит описание пользовательского интерфейса
- `docs/ai/usecase/*.md` содержит описание сценариев использования.
- `docs/ai/userstory/*.md` содержит описание пользовательских сценариев.


## Игнорирование файлов
- `.gitignore` — стандартный гитигнор (build, node_modules, .env, IDE, OS).
- `.llmignore` — паттерны файлов, которые агент не должен генерировать и коммитить (scratch, backups, generated stubs).

## Запрещённые для чтения файлы
- `docs/ai/prompts_history.md` — файл с историей промптов, агенту запрещено читать этот файл.

## Описание проекта
- Мультимодульный проект
- Сборщик проекта - Gradle
- В каждом модуле может быть свой `AGENTS.md` со специфичными для каждого модуля ограничениями.

# Перезапуск в докере
Пользуемся секцией develop watch из docker compose, запускаем его в режиме слежения и просто пересобираем необходимые артефакты
